# STAiR 스테어 커피 · 매장용 대시보드

사이폰 카페 **스테어**의 메뉴를 손님이 **아이패드**로 직접 보면서 원두의 특징과 사장님의 한마디를 확인하고, 주문까지 내릴 수 있는 **단일 파일 웹앱**입니다. 별도 서버나 빌드 도구 없이 브라우저에서 바로 실행됩니다.

---

## 1. 파일 구성

```
index.html     ← 앱 본체 (React + Tailwind CDN, 단일 파일)
README.md                 ← 이 문서 (구동/배포 가이드)
stair_admin_export.json   ← (선택) 관리자 탭에서 내보낸 백업
```

유지보수는 `index.html` 한 파일만 편집하면 됩니다. 핵심 영역은 다음과 같습니다.

| 섹션 | 위치 (파일 내) | 하는 일 |
|---|---|---|
| `CATALOG = { ... }` | 파일 상단 `<script>` 블록 | 메뉴(원두/디저트/차/음료) 데이터 |
| `defaultOwnerNotes` | `CATALOG` 바로 아래 | 최초 배포 시의 "사장님 한마디" |
| `defaultGreeting` | 같은 블록 | 메인 페이지 인사말 |
| `pw==='stair'` (2곳) | `AdminView` 내부 | 관리자 비밀번호 |
| `<style>` 블록 | `<head>` 내부 | 색/타이포(페이퍼/골드 팔레트) |

## 3. 구동 방법

### (1) 매장 아이패드에서 띄우기 — 추천
1. `index.html`을 아이패드의 **파일** 앱에 저장.
2. **Safari**로 해당 파일을 연다.
3. 주소창 공유 버튼 → **홈 화면에 추가** → 전체화면 앱처럼 사용.
4. (선택) 설정 → 손쉬운 사용 → **가이드 액세스**로 앱을 잠가두면 손님이 다른 화면으로 넘어가지 않습니다.

### (2) PC/노트북에서 바로 실행
- `index.html` 파일을 **더블클릭**하면 기본 브라우저에서 열립니다.

### (3) 같은 와이파이의 여러 기기에서 공유
1. 파일을 한 대의 컴퓨터에 두고 터미널에서:
   ```
   cd <이 파일이 있는 폴더>
   python3 -m http.server 8080
   ```
2. 다른 기기(아이패드 등)의 Safari에서 `http://<컴퓨터 IP>:8080/index.html` 접속.

### (4) 인터넷 배포(선택)
- 정적 파일 하나만 올리면 되므로 **GitHub Pages / Netlify / Vercel**에 드래그 앤 드롭 배포가 가능합니다.

## 4. 데이터 저장 · 백업

- 사장님 한마디, 메인 인사말, 주문 기록은 **해당 브라우저의 `localStorage`**에 저장됩니다.
- 기기나 브라우저를 바꾸려면 **관리자 탭 → 내보내기(JSON)** 로 백업 → 새 기기에서 **가져오기**.
- 공장 초기값으로 되돌리려면 관리자 탭의 **초기화** 버튼 사용.

## 5. 유지보수 가이드 (자주 하는 작업)

### 메뉴 추가/수정
`index.html` 안의 `CATALOG` 객체에서 해당 섹션 배열에 항목을 추가하세요.

```js
{
  id: 'b12',                         // 고유 ID (충돌 금지)
  name: '엘살바도르 엘 카르멘 카투아이 허니',
  price: 10000,
  origin: '엘살바도르 / 아파네카',
  variety: 'Catuai',
  process: 'Honey (허니)',
  altitude: '1,500m',
  body: 3, acid: 3, sweet: 5, balance: 4,
  notes: ['브라운슈거', '배', '호두'],
  whenMoods: ['편안한 오후', '가벼운 당 충전'],
  tagline: '부드럽고 달콤한 허니 프로세스',
  expert: '허니 프로세스 특유의 은은한 단맛. 입문자에게도 부담 없습니다.',

```

### 가격 일괄 변경
같은 파일에서 `price: NNNN` 값을 직접 수정하세요.

### 사장님 한마디 초기값 변경
`defaultOwnerNotes`의 `b1 ~ b11`, `dc1`, `bl1` 값을 편집.
(이후 새 브라우저에서 최초 열릴 때 적용됩니다. 이미 사용 중인 기기는 관리자 탭 → 초기화가 필요합니다.)

### 비밀번호 변경
`AdminView` 함수 안의 `pw==='stair'` 두 곳을 원하는 비밀번호로 바꿉니다.

### 테마 색 변경
`<style>` 블록 상단의 `:root` CSS 변수(`--paper`, `--ink`, `--gold` 등) 값 교체.

## 6. 아키텍처 개요

- **런타임**: 브라우저 단일 페이지 앱 (SPA). React 18 UMD 번들 + Babel standalone으로 JSX를 브라우저에서 컴파일, Tailwind CDN으로 스타일.
- **상태 관리**: React `useState` + `localStorage` 영속화(하나의 저장 키 `stair_v1_state`).
- **데이터 모델**:
  - `CATALOG`  (읽기 전용, 정적)
  - `state = { notes, greeting, orders }`  (편집 가능, 저장됨)
  - `cart = [ { id, name, basePrice, options:{iced,roast,subtype}, qty } ]` (세션 전용)
- **UI 규칙**:
  - 손님 화면(메뉴 탭)에는 절대 관리자 기능 노출 X.
  - 원두 카드 → 상세 모달 → 옵션 선택 → 카트 반영 흐름으로 통일.
  - iPad 세로/가로 모두 동작하도록 반응형 그리드 사용.
